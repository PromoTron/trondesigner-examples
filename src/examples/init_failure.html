Init Failure Handling

This example intentionally uses an invalid API key (INVALID_KEY_TO_TRIGGER_FAILURE) so that the SDK raises a scriptInitFailed event on load.

The data-td-script-init-failed-callback attribute points at onScriptInitFailed, which displays a modal popup with the error message. The designer button is shown as disabled to make it clear the integration is unavailable.

Callback attributes used here:

  • data-td-script-initialized-callback
  • data-td-script-init-failed-callback

Open Designer (will fail — invalid key)

⚠️

Initialization Failed

The TronDesigner SDK could not initialize. Please check your API key and try again.

Implementation Code

<!-- Use a wrong key to force the failure callback -->
<script
    async
    src="{TD_URL}/js/website-integration.js"
    data-td-api="{TD_URL}/api"
    data-td-api-key="INVALID_KEY_TO_TRIGGER_FAILURE"
    data-td-script-initialized-callback="onScriptInitialized"
    data-td-script-init-failed-callback="onScriptInitFailed">
</script>

<script>
    function onScriptInitFailed(error) {
        const message = error && error.message ? error.message : String(error);
        document.getElementById('error-message').textContent = message;
        document.getElementById('error-overlay').classList.add('visible');
    }

    function closeNotification() {
        document.getElementById('error-overlay').classList.remove('visible');
    }
</script>

<!-- Modal popup, hidden until init fails -->
<div id="error-overlay" class="notification-overlay">
    <div class="notification-popup">
        <h2>Initialization Failed</h2>
        <p>The TronDesigner SDK could not initialize.</p>
        <p><code id="error-message"></code></p>
        <button onclick="closeNotification()">Dismiss</button>
    </div>
</div>
