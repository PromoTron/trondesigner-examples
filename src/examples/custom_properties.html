Custom Properties

Custom properties are arbitrary key/value metadata you attach to a print job. TronDesigner stores them unchanged and echoes them back in every callback and webhook — letting your backend correlate completed designs with cart lines, campaigns, or any internal context.

The same customProperties object can be passed in three ways. All three produce identical behaviour; pick the one that fits your integration style:

  • HTML data attribute — data-td-custom-properties JSON string on the action button
  • JS builder — mutate the object returned by argsBuilder.build() before passing it to openDesigner()
  • Print job arguments — nest customProperties directly inside the printJob argument of openDesigner()

After you finish a design, the printJob.created callback fires. The custom properties you sent are highlighted in green below the event log.

Implementation Code


    
1 — HTML data attribute

    
<button
    data-td-action="OpenCreate"
    data-td-supplier-code="PFC"
    data-td-product-code="10086021"
    data-td-print-position-code="front"
    data-td-print-technology-code="PAD03"
    data-td-custom-properties='{"source":"html-attr","campaign":"SPRING24","region":"EU"}'>
    Open Designer
</button>


    
2 — JS builder

    
const td = window.tronDesigner;
td.setExactProduct('PFC', undefined, (b) => { b.addVariant('10086021'); });
td.addExactPrinting('front', 'PAD03');

const args = td.argsBuilder.build();
args.printJob.customProperties = {
    source: 'js-builder',
    cartLineId: 'CL-1234',
    campaign: 'SPRING24'
};

td.openDesigner(args, { mode: 'dialog' });


    
3 — Print job arguments

    
window.tronDesigner.openDesigner({
    openMode: 'Create',
    dataMode: 'Exact',
    printJob: {
        addPrintings: [{ printPositionCode: 'front', printTechnologyCode: 'PAD03' }],
        addProducts: {
            supplierCode: 'PFC',
            variants: [{ productCode: '10086021' }]
        },
        customProperties: {
            source: 'printjob-args',
            cartLineId: 'CL-1234',
            campaign: 'SPRING24'
        }
    }
}, { mode: 'dialog' });


    
Receiving custom properties in the created callback

    
<!-- Wire the callback via script-element attribute -->
<script
    async
    src="{TD_URL}/js/website-integration.js"
    data-td-api="{TD_URL}/api"
    data-td-api-key="{TD_API_KEY}"
    data-td-print-job-created-callback="onPrintJobCreated">
</script>

<script>
function onPrintJobCreated(event) {
    // event.entityGuid  — the new print job GUID
    // event.detail.customProperties  — your metadata echoed back unchanged
    const customProps = event.detail.customProperties;
    console.log('Custom properties received:', customProps);
}
</script>
