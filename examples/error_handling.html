Error Handling & Validation

Demonstrates proper error handling techniques for TronDesigner integration, including validation, error recovery, and user feedback.

Best Practices:

  • Pre-flight validation of configuration parameters
  • Try-catch blocks for synchronous errors
  • Promise catch handlers for async errors
  • User-friendly error messages
  • Logging for debugging
  • Graceful degradation and retry logic

Error Handling Patterns

1. Pre-flight Validation

if (!productCode || !supplierCode) {
    showError("Missing required parameters");
    return;
}

2. Try-Catch for Sync Errors

try {
    td.openDesigner(config, options);
} catch (error) {
    console.error("Failed to open:", error);
    showError(error.message);
}

3. Promise Catch for Async Errors

td.openDesigner(config, options)
    .then(() => console.log("Success"))
    .catch((error) => {
        console.error("Error:", error);
        showError(error.message);
    });

4. Timeout Handling

const timeoutPromise = new Promise((_, reject) => {
    setTimeout(() => reject(new Error("Timeout")), 5000);
});

Promise.race([openPromise, timeoutPromise])
    .then(() => console.log("Success"))
    .catch((error) => console.error(error));
