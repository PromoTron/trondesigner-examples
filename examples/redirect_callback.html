Advanced Result Actions: REDIRECT & CALLBACK

Beyond the basic result actions (NONE, CLOSE, DIALOG), you can redirect users to a custom URL or execute JavaScript callbacks after print job completion.

Result Action Types:

  • REDIRECT - Navigate to a custom URL with print job data in query parameters
  • CALLBACK - Execute a named JavaScript function with print job data
  • Alternative: Use subscribeDesignerEvent() for more control over callbacks

Implementation Examples

1. REDIRECT Result Action

"editor": {
    "result": {
        "resultAction": "REDIRECT",
        "redirectUrl": "https://yoursite.com/cart?printJobGuid={printJobGuid}"
    }
}

The {printJobGuid} placeholder will be replaced with the actual print job GUID.

2. CALLBACK Result Action

// Define callback function
window.onPrintJobComplete = function(data) {
    console.log("Print job completed:", data.printJobGuid);
    // Add to cart, update UI, etc.
};

// Configure result action
"editor": {
    "result": {
        "resultAction": "CALLBACK",
        "callbackFunction": "onPrintJobComplete"
    }
}

3. Event-Based Approach (Recommended)

td.subscribeDesignerEvent("printJob", "created", (data) => {
    console.log("Print job created:", data.printJobGuid);

    // Your custom logic here
    addToCart(data.printJobGuid);
    showSuccessMessage();
    updateAnalytics(data);
});

Use Cases

  • REDIRECT: E-commerce cart flow, order confirmation pages, thank you pages
  • CALLBACK: Single-page apps, real-time UI updates, analytics tracking
  • Events: Complex workflows, multiple actions, conditional logic
